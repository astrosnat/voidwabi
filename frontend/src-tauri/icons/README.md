# Icon Setup Guide

This directory contains icon files for the Tauri application across all platforms.

## Current Icons

- **icon.ico** (126 KB) - Windows icon with multiple resolutions (16, 32, 48, 64, 128, 256px)
- **icon.png** - Generated fallback PNG icon
- **Additional sizes** - Generated automatically by Tauri build scripts

  - `256x256.png`
- **Status**: Generated by setup script from source

## Source Icon

The project uses **app-icon.png** (1024x1024 RGBA PNG) as the source for automatic icon generation.

### Generating Icons Automatically (Recommended)

From the `frontend/` directory:
```bash
tauri icon ../app-icon.png
```

This will automatically generate all required icons:
- Windows `.ico` with all layers
- macOS `.icns` with all layers
- Linux PNG files at various sizes

## Manual Icon Replacement

To replace the icon:

1. **Option A: Update source PNG**
   - Replace `app-icon.png` in project root with your 1024x1024 RGBA PNG
   - Run: `tauri icon ../app-icon.png` from `frontend/` directory

2. **Option B: Update ICO directly**
   - Replace `icon.ico` with your Windows icon file
   - Ensure it contains layers at: 16, 32, 48, 64, 256px

## Build Process

The GitHub Actions workflow automatically:
1. Runs the icon setup script (`scripts/setup-icons.js`)
2. Generates missing icon files from available sources
3. Falls back to copied/temporary icons if generation fails
4. Bundles appropriate icons for each target platform

## Troubleshooting

**Build fails with "icon not found":**
- Ensure `icon.ico` exists in this directory
- Run: `node scripts/setup-icons.js` locally
- Check that icon files are not corrupted

**Icons look blurry/wrong:**
- Verify `icon.ico` has proper layers at all required sizes
- Regenerate using: `tauri icon ../app-icon.png`
- Use 32px layer as primary (should be first in file)

**Platform-specific icon issues:**
- Windows: `.ico` format required, check layer order
- macOS: `.icns` format required, 10 layers needed
- Linux: RGB/RGBA PNG files at standard sizes
